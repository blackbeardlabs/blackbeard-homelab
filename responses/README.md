# Responses

Store model responses from benchmark runs here when the output is useful for later quality review.

Use one Markdown file per recorded response. Keep benchmark metrics in `benchmarks/` and raw backend logs in `raw-logs/`; this folder is for the model's actual answer.

## Notes

- Preserve the response as closely as possible.
- Add human notes separately from the response.
- Do not rewrite responses to fix quality issues.
- If the response is very large, summarize the storage decision in the benchmark file and keep the full response here.
